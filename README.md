# Parabank_Web_Testing_Project

## 📌 Overview
This project focuses on end-to-end testing of the Parabank web application, covering core banking functionalities through manual testing and basic API validation.
The goal was to simulate a real-world QA workflow by planning, executing, and reporting testing activities in a structured manner.

## 🎯 Objectives
- Validate core banking features and user flows
- Identify functional and usability issues
- Perform structured manual testing
- Conduct basic API testing using Postman
- Document defects and summarize findings

## 🧩 Features Tested
### 🔹 Web Application (Manual Testing)
- User Registration
- Login
- Account Creation
- Fund Transfer
- Bill Payment
- Transactions
- Loan Request
- Customer Care
- Logout

### 🔹API Testing (Postman)
- Retrieve Transactions
- Bill Payment
- Deposit
- Transfer
- Withdraw
- Login
- Loan Request

⚠️ Note: API testing was performed at a basic level to validate functionality and responses.

## 🛠️ Testing Approach
### ✔ Manual Testing
- Test Plan creation
- Test Scenarios (feature-based)
- Test Case design and execution
- Bug reporting with severity and priority

### ✔ API Testing
- Tested endpoints using Postman
- Verified status codes and response behavior
- Covered basic positive and negative scenarios

  - Manual: 29
  - API: 12

## 🐞 Defect Summary
- Total Bugs Found: 5
- High Severity: 4
- Medium Severity: 1
- Low Severity: 0

## 📌 Key Findings
- Missing validation messages in some flows
- Incorrect behavior in bill payment and loan request edge cases
- Minor UI inconsistencies across pages

## ⚠️ Risks / Limitations
- Limited test data availability
- Inconsistent input validation behavior
- Some features were not fully testable

## 📄 Deliverables
1. Test Plan
2. Test Scenarios (Manual & API)
3. Test Cases (Manual & API)
4. Test Execution Report
5. Bug Reports
6. Test Summary Report

## 🧾 Conclusion
The application is functional for basic use; however, several high-severity issues were identified that impact reliability and user experience. These should be addressed before production release.

## 🚀 Key Learnings
- Developed structured QA workflow from planning to reporting
- Improved ability to identify edge cases and validation issues
- Gained hands-on experience with API testing using Postman
- Strengthened bug reporting and analytical thinking skills
